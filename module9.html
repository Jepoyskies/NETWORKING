Module 9: FHRP Concepts

Goal: Learn how to keep the network running even if a router dies.

9.1 First Hop Redundancy Protocols

Default Gateway Limitations

End devices are configured with a single default gateway. If that physical router fails, the PCs lose outside connectivity.

First hop redundancy protocols (FHRPs) provide alternate gateways by allowing multiple routers to share a single virtual router IP and MAC address.

Steps for Router Failover

In FHRP, when computers use ARP to find the gateway, the routers reply with the Virtual MAC address.

  1. The standby router stops seeing Hello messages from the forwarding router.
  2. The standby router assumes the active role.
  3. Host devices see absolutely no disruption in service!

9.2 HSRP (Hot Standby Router Protocol)

HSRP Priority and Preemption

In HSRP, the router with the highest priority becomes the active router. (Default priority is 100).

If the Active router fails, the Standby takes over. If the original Active router gets fixed and boots back up, it will NOT take its job back by default. To force it to take its job back, preemption must be enabled using the standby preempt command.

HSRP States

A router booting up with HSRP goes through these states:

  • Initial: Just turned on.
  • Learn: Waiting to hear from the active router.
  • Listen: Knows the virtual IP, but is neither active nor standby.
  • Speak: Sending Hello messages, participating in the election.
  • Standby: The candidate ready to take over next.
  • Active: Currently forwarding the traffic.
