ITCC 32.1: IPv6 DHCPv6 Lab Simulator

Master Stateless DHCPv6, Stateful DHCPv6, and DHCP Relay Agents!

Lab Objectives

  • Enable ipv6 unicast-routing globally on both routers.
  • R1 (Stateless DHCPv6): Configure pool R1-STATELESS with DNS 2001:db8:acad::254 and domain STATELESS.com.
  • Apply it to R1's g0/0/1 interface and set the O-flag (other-config-flag).
  • R1 (Stateful DHCPv6): Configure pool R2-STATEFUL with address prefix 2001:db8:acad:3:aaa::/80, DNS 2001:db8:acad::254, and domain STATEFUL.com.
  • Apply it to R1's g0/0/0 interface.
  • R2 (DHCP Relay): Go to R2's g0/0/1 interface, set the M-flag (managed-config-flag), and set the DHCP Relay destination to R1's G0/0/0 IP (2001:db8:acad:2::1).

📜 Cisco CLI Cheat Sheet: Mod 8 DHCPv6

1. Basic IPv6 Routing

ipv6 unicast-routing
Must be enabled globally before a router can send Router Advertisement (RA) messages!
ipv6 route ::/0 2001:db8:acad:2::2
Configures an IPv6 default static route.

2. Stateless DHCPv6 (SLAAC + DHCPv6 for DNS)

ipv6 dhcp pool R1-STATELESS
dns-server 2001:db8:acad::254
domain-name STATELESS.com
Creates a pool that ONLY hands out DNS info (no IP addresses).
interface g0/0/1
ipv6 nd other-config-flag
ipv6 dhcp server R1-STATELESS
Changes the RA message O-flag to 1 (telling PCs to use SLAAC for IPs, but DHCPv6 for DNS) and binds the pool to the interface.

3. Stateful DHCPv6

ipv6 dhcp pool R2-STATEFUL
address prefix 2001:db8:acad:3:aaa::/80
dns-server 2001:db8:acad::254
domain-name STATEFUL.com
Creates a pool that hands out FULL IPv6 addresses AND DNS info.

4. DHCPv6 Relay Agent (IP Helper for IPv6)

interface g0/0/1
ipv6 nd managed-config-flag
ipv6 dhcp relay destination 2001:db8:acad:2::1 g0/0/0
Changes the RA message M-flag to 1 (telling PCs to get their IP from DHCPv6) and forwards those DHCP requests to R1's IP address.
