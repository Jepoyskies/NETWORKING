Module 8: SLAAC and DHCPv6

Goal: Learn how IPv6 addresses are handed out automatically.

8.1 IPv6 GUA Assignment

IPv6 Host Configuration

On a router, an IPv6 global unicast address (GUA) is manually configured. But for Windows PCs, typing long IPv6 addresses manually is prone to errors. Therefore, most hosts acquire them dynamically.

IPv6 Host Link-Local Address

If automatic addressing is selected, the host relies on an ICMPv6 Router Advertisement (RA) message from the router.

The IPv6 link-local address is automatically created by the host when it boots up. (You will see it starts with fe80::).

Three RA Message Flags

How a client obtains its IPv6 address depends on 3 flags set inside the router's RA message:

  • A flag (Autoconfiguration) - Tells the PC: "Use SLAAC to create your own IPv6 address."
  • O flag (Other Configuration) - Tells the PC: "Get your IP from SLAAC, but seek extra info (like DNS) from a stateless DHCPv6 server."
  • M flag (Managed Address) - Tells the PC: "Do NOT use SLAAC. Use a stateful DHCPv6 server to get your IP."

8.2 SLAAC (Stateless Address Autoconfiguration)

SLAAC Overview

Not every network has a DHCPv6 server. SLAAC enables hosts to create their own unique IPv6 GUA without one.

It is stateless, meaning there is no server tracking who has what IP. SLAAC relies on the router sending periodic RA messages every 200 seconds. A host can also send an RS message requesting an RA immediately.

Generating the Interface ID & DAD

With SLAAC, the router gives the 64-bit network prefix. The PC must generate the remaining 64-bit Interface ID using either:

  • Randomly Generated: Used by Windows 10 for privacy.
  • EUI-64: Uses the PC's MAC address and inserts fffe in the middle.

To ensure the newly created IP address is completely unique, the host performs Duplicate Address Detection (DAD) before using it.

8.3 & 8.4 DHCPv6 Operation

DHCPv6 Operation Steps

  1. Host sends an RS message.
  2. Router responds with an RA message indicating DHCPv6 should be used.
  3. Host sends a DHCPv6 SOLICIT message.
  4. DHCPv6 server responds with ADVERTISE.
  5. Host responds with REQUEST to the server.
  6. Server sends REPLY message.

Router Roles

A Cisco router can act as:

  • DHCPv6 Server: Provides stateless or stateful services.
  • DHCPv6 Client: Obtains an IP from another server.
  • DHCPv6 Relay Agent: Forwards DHCP messages if the server is on a different network. Command: ipv6 dhcp relay destination [IP].
