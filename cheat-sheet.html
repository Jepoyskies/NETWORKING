📜 Master CLI Cheat Sheet

A complete reference guide for all router and switch commands organized by technology. Tap any blue dashed word for definitions!

1. Basic Device & Interface Setup

Set Hostname & Passwords

hostname R1
enable secret class
service password-encryption
Changes device name, secures Privileged EXEC mode, and scrambles all plaintext passwords.

Secure Console & Banners

banner motd # Authorized Personnel Only #
line console 0
password cisco
login
logging synchronous
Sets a warning banner and locks down the physical console port.

Basic Interface Commands

interface g0/1
description Connects to Sales LAN
no shutdown
Adds a helpful label and manually turns on a physical interface.

Save Configuration

copy running-config startup-config
Saves the active RAM config into NVRAM so it survives a reboot.

2. Verification 'Show' Commands

show ip interface brief
Displays a quick list of all ports, their IP addresses, and if they are Up or Down.
show vlan brief
Lists all VLANs created on the switch and which ports belong to them.
show interfaces trunk
Verifies which ports are actively trunking and what the Native VLAN is.
show etherchannel summary
Checks if your bundled ports successfully formed an EtherChannel (Look for 'SU' status).
show ip route
Displays the routing table to verify connected networks.
show ip dhcp binding
Shows a list of all IP addresses the server successfully handed out to PCs.
show standby
Displays detailed HSRP info (Active/Standby IPs, Virtual MAC address, timers).
show port-security interface f0/1
Checks port security status, violation mode, and max MACs for a specific port.

3. IP Addressing & DHCP Server

Assign IPv4 & IPv6 Addresses

interface g0/0
ip address 192.168.1.1 255.255.255.0
ipv6 address 2001:db8:acad:1::1/64
ipv6 address fe80::1 link-local
Assigns standard IPv4, global IPv6, and a specific IPv6 link-local address.

Layer 2 Switch IP & Gateway

interface vlan 1
ip address 192.168.1.2 255.255.255.0
no shutdown
exit
ip default-gateway 192.168.1.1
Gives a switch an IP and default gateway for remote management.

Configure a DHCP Server (IPv4)

ip dhcp excluded-address 192.168.1.1 192.168.1.10
ip dhcp pool LAN-POOL
network 192.168.1.0 255.255.255.0
default-router 192.168.1.1
Reserves the first 10 IPs, then creates a DHCP pool assigning IPs and gateways to PCs.

DHCP Helper (Relay)

interface g0/1
ip helper-address 10.1.1.5
Forwards local DHCP broadcasts across a router to a centralized remote server.

4. VLANs, Routing & L3

Access Ports & Voice VLANs

vlan 10
name Admin
vlan 50
name Voice
interface range f0/1 - 5
switchport mode access
switchport access vlan 10
switchport voice vlan 50
mls qos trust cos
Creates VLANs. Assigns PCs to VLAN 10, IP Phones to VLAN 50, and tells the switch to trust the phone's QoS priority.

Router-on-a-Stick (Inter-VLAN)

interface g0/0.10
encapsulation dot1Q 10
ip address 192.168.10.1 255.255.255.0
Creates a virtual subinterface, maps it to VLAN 10 using 802.1Q tags, and assigns the gateway IP.

Enable Layer 3 Switch Routing

ip routing
ipv6 unicast-routing
Turns on the routing engine inside a Multilayer Switch so it can route between VLANs.

Convert L2 Port to L3 Port

interface g0/2
no switchport
ip address 10.1.1.1 255.255.255.252
Removes switching capabilities from a port so you can assign a direct IP address like a normal router.

5. Advanced Switch Security

Secure Trunks (Mitigate VLAN Hopping)

interface range g0/1 - 2
switchport mode trunk
switchport nonegotiate
switchport trunk native vlan 100
Hardcodes the trunk, turns off DTP negotiation to prevent spoofing, and sets a dedicated Native VLAN.

Secure Unused Ports (BlackHole VLAN)

vlan 999
name BlackHole
exit
interface range f0/3 - 9, f0/11 - 23
switchport mode access
switchport access vlan 999
shutdown
Best practice: Create a dead-end VLAN, move all unused physical ports into it, and manually shut them down.

Port Security (Mitigate MAC Flooding)

interface range f0/1 - 2
switchport mode access
switchport port-security
switchport port-security maximum 4
switchport port-security mac-address sticky
switchport port-security violation restrict
Restricts active ports to a set number of MACs, learns them dynamically (sticky), and drops/logs bad traffic (Restrict mode keeps the port UP).

DHCP Snooping

ip dhcp snooping
ip dhcp snooping vlan 10,20,99
interface range g0/1 - 2
ip dhcp snooping trust
interface range f0/1 - 2
ip dhcp snooping limit rate 5
Enables snooping globally and per-VLAN, trusts the uplinks to the actual server, and limits user ports to 5 packets/sec to stop DHCP Starvation.

Spanning Tree Edge Security

spanning-tree mode rapid-pvst
spanning-tree portfast default
interface range f0/1 - 2
spanning-tree bpduguard enable
Speeds up convergence with Rapid PVST+. Enables PortFast on all access ports, and protects them with BPDU Guard to instantly kill the port if a rogue switch is plugged in.

6. HSRP & Advanced IPv6

HSRP Configuration

interface g0/1
standby version 2
standby 1 ip 192.168.1.254
standby 1 priority 150
standby 1 preempt
Configures the Active Router in an HSRP group.

Stateless DHCPv6 (SLAAC + DNS)

ipv6 dhcp pool STATELESS-POOL
dns-server 2001:db8::10
!
interface g0/0
ipv6 nd other-config-flag
ipv6 dhcp server STATELESS-POOL
Sets the O-Flag in the RA to tell clients to use SLAAC for their IP but ask a DHCPv6 server for DNS.

Stateful DHCPv6

ipv6 dhcp pool STATEFUL-POOL
address prefix 2001:db8:1::/64
dns-server 2001:db8::10
!
interface g0/0
ipv6 nd managed-config-flag
ipv6 dhcp server STATEFUL-POOL
Sets the M-Flag in the RA to force clients to get their IP and DNS directly from the DHCPv6 server.

The Hidden Exam Traps

Trap: Router as DHCP Client

interface g0/0/1
ip address dhcp

Forces a router to request an IP from a DHCP server.

7. Wireless LAN (WLC & APs)

Lightweight AP Setup Process

1. Power on AP (Needs physical cord in PT)
2. AP grabs IP via DHCP on the switch
3. AP searches for WLC automatically
4. AP downloads config (Zero touch)
Remember: LAPs have "no brains"! You do NOT click the AP to configure it. You configure everything via an Admin PC web browser.

WLC GUI: RADIUS Security Server

SECURITY Tab → RADIUS → Authentication
Click New
Server IP: [IP Address of AAA Server]
Shared Secret: [Password]
Tells the WLC where the Enterprise database is located so it can verify usernames and passwords.

WLC GUI: Create & Enable WLAN

WLANs Tab → Create New → Name & SSID
General Tab → Status: [x] Enabled
Security Tab → L2: WPA+WPA2 / 802.1X
Security Tab → AAA: Select Server IP
Creates the wireless network. The SSID is the name broadcasted to devices.

The Hidden Exam Traps

Trap: Missing HTTPS

You must explicitly type https://[WLC-IP] in the PC web browser. If you just type the IP, it will fail.

Trap: WLAN Won't Broadcast

By default, new WLANs are disabled. You MUST check the Status: [x] Enabled box on the General Tab.
