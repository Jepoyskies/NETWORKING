Module 11 Quiz: Switch Security

1. What is a recommended best practice when dealing with the native VLAN?
Correct Answer: Assign it to an unused VLAN.

Deep Dive: The Native VLAN is responsible for carrying untagged traffic across a trunk link. Hackers exploit the default Native VLAN (VLAN 1) to launch Double-Tagging VLAN Hopping attacks. The industry best practice is to change the Native VLAN on all trunk links to a dedicated, unused "black hole" VLAN ID (e.g., VLAN 999) that has no host devices assigned to it.

  • A. Turn off DTP: (Incorrect) This stops basic VLAN Hopping, but does not address Native VLAN Double-Tagging.
  • B. Same as management: (Incorrect) The management VLAN should be kept strictly separate from the native VLAN.
  • D. Assign to an unused VLAN: (Correct) Isolates untagged traffic into a dead-end VLAN.
2. On what switch ports should PortFast be enabled to enhance STP stability?
Correct Answer: All end-user ports

Deep Dive: PortFast forces a switch port to immediately skip the 30-second STP listening and learning phases, going straight to the forwarding state. This feature is designed *exclusively* for edge ports connected to end-user devices (PCs, servers, printers). If you mistakenly enable PortFast on a link connected to another switch, you risk creating an instant Layer 2 broadcast storm.

3. Which command would be best to use on an unused switch port if a company adheres to the best practices as recommended by Cisco?
Correct Answer: shutdown

Deep Dive: Unlike Cisco routers, switch ports are enabled (turned on) by default right out of the box. If a wall jack in a conference room is not currently needed, the absolute simplest and most effective security measure is to administratively disable it using the shutdown command. This completely prevents rogue devices from physically linking to the network.

  • A, B, C: (Incorrect) These are advanced security commands intended for ports that are *actively in use* by legitimate employees. For an *unused* port, just turn it off!
  • D. shutdown: (Correct) Disables the interface entirely.
4. Which two features on a Cisco Catalyst switch can be used to mitigate DHCP starvation and DHCP spoofing attacks? (Combined)
Correct Answer: Port security AND DHCP snooping

Deep Dive: You need two tools to fix these two attacks.
1. DHCP Starvation involves a hacker spoofing thousands of MAC addresses. You stop this using Port Security (which restricts a port to 1 or 2 MAC addresses).
2. DHCP Spoofing involves a hacker acting as a fake DHCP server. You stop this using DHCP Snooping (which filters out DHCP server messages coming from untrusted user ports).

5. What is the best way to prevent a VLAN hopping attack?
Correct Answer: Disable trunk negotiation for trunk ports and statically set nontrunk ports as access ports.

Deep Dive: Hackers execute a basic VLAN Hopping attack (Switch Spoofing) by sending fake DTP messages to a switch, tricking it into giving the hacker a trunk link. To completely eliminate this threat, you must statically define your ports. Hardcode user ports with switchport mode access, hardcode switch uplinks with switchport mode trunk, and explicitly turn off DTP negotiation using switchport nonegotiate.

6. Which procedure is recommended to mitigate the chances of ARP spoofing?
Correct Answer: Enable DHCP snooping on selected VLANs.

Deep Dive: To stop ARP Spoofing (ARP Poisoning), you must use Dynamic ARP Inspection (DAI). However, DAI doesn't work by itself! It acts as a bouncer, checking IDs against a list. That list is the "DHCP Snooping Binding Table." Therefore, the very first prerequisite procedure to mitigating ARP spoofing is that you MUST enable DHCP Snooping globally and on the selected VLANs to build that database!

  • A. Enable DAI on mgmt VLAN: (Incorrect) DAI should be on user VLANs, not just management.
  • C. Enable DHCP snooping on selected VLANs: (Correct) This is the mandatory prerequisite foundation that makes DAI possible.
7. What are two types of switch ports that are used on Cisco switches as part of the defense against DHCP spoofing attacks? (Combined)
Correct Answer: Trusted DHCP port AND Untrusted port

Deep Dive: DHCP Snooping classifies every port on a switch into two categories. Untrusted ports face end-users; the switch will instantly drop any DHCP Server (Offer/Ack) messages originating from them. Trusted ports are the uplinks that physically lead toward your legitimate enterprise DHCP server.

8. Which two commands can be used to enable PortFast on a switch? (Combined)
Correct Answer: S1(config)# spanning-tree portfast default AND S1(config-if)# spanning-tree portfast

Deep Dive: PortFast can be enabled globally or per-interface.
1. Global: spanning-tree portfast default (turns it on automatically for all non-trunking access ports).
2. Interface: spanning-tree portfast (turns it on manually for the specific interface you are currently inside).

9. An administrator who is troubleshooting connectivity issues on a switch notices that a switch port configured for port security is in the err-disabled state. After verifying the cause of the violation, how should the administrator re-enable the port without disrupting network operation?
Correct Answer: Issue the shutdown command followed by the no shutdown command on the interface.

Deep Dive: When Port Security triggers a violation, it places the port into an err-disabled state (a forced, secure coma). You cannot simply type `no shutdown` to fix it. The switch requires you to acknowledge the event by manually resetting the port. You must explicitly type shutdown to clear the error state, and then type no shutdown to bring the port back to life.

10. A network administrator is configuring DHCP snooping on a switch. Which configuration command should be used first?
Correct Answer: ip dhcp snooping

Deep Dive: Cisco IOS commands are hierarchical. Before you can configure specific DHCP Snooping interface trusts, rate limits, or VLAN scopes, you must activate the master switch that turns the entire feature on globally! That very first command, typed in global config mode, is simply ip dhcp snooping.

11. A network administrator is configuring DAI on a switch with the command ip arp inspection validate dst-mac. What is the purpose of this configuration command?
Correct Answer: To check the destination MAC address in the Ethernet header against the target MAC address in the ARP body

Deep Dive: Advanced DAI (Dynamic ARP Inspection) can validate the payload of the ARP packet itself. Hackers try to spoof systems by putting different data in the outer envelope than in the inner letter. The validate dst-mac command forces the switch to compare the Destination MAC on the outer Ethernet header directly against the Target MAC listed inside the ARP payload. If they don't match, the packet is dropped.

12. Which security feature should be enabled in order to prevent an attacker from overflowing the MAC address table of a switch?
Correct Answer: Port security

Deep Dive: A MAC address table overflow attack floods a switch port with thousands of spoofed source MAC addresses. By configuring Port Security and setting a maximum allowed MAC limit (like 1 or 2), the switch will block the port the moment the hacker tries to send a barrage of fake addresses.

  • A. Storm control: (Incorrect) Prevents excessive broadcast traffic, but doesn't track MAC table limits.
  • B. Port security: (Correct) Limits the number of MACs allowed per physical port.
  • C & D. Root guard / BPDU filter: (Incorrect) These protect Spanning Tree Protocol.
13. What Layer 2 attack is mitigated by disabling Dynamic Trunking Protocol?
Correct Answer: VLAN hopping

Deep Dive: Dynamic Trunking Protocol (DTP) automatically forms trunk links if a switch port detects another switch on the other side. Hackers exploit this by running software that sends fake DTP messages, tricking the port into becoming a trunk. The hacker can then access every VLAN on the network! Disabling DTP with `switchport nonegotiate` completely stops this VLAN Hopping attack.

14. A network administrator is configuring DAI on a switch. Which command should be used on the uplink interface that connects to a router?
Correct Answer: ip arp inspection trust

Deep Dive: When you enable Dynamic ARP Inspection (DAI), all ports become "Untrusted" by default, meaning they will rigorously inspect and potentially drop ARP packets. Uplinks that connect to routers or core switches are generally safe and process massive amounts of legitimate ARP traffic. You must explicitly bypass inspection on these uplinks using the ip arp inspection trust interface command to prevent accidental drops.

15. Where are dynamically learned MAC addresses stored when sticky learning is enabled with the switchport port-security mac-address sticky command?
Correct Answer: RAM

Deep Dive: When a switch dynamically learns a Sticky MAC address, it literally pastes that address as a configuration line into the active running configuration. Because the running-config is volatile and actively operating the switch, it is stored in RAM. If the switch loses power, the RAM (and the sticky MACs) are wiped clean unless the admin types `copy run start` to save it to NVRAM.
