Module 11: Switch Security Configuration

Goal: Learn the exact commands and configurations to lock down Layer 2 switches against LAN attacks.

11.1 Implement Port Security

Secure Unused Ports

Layer 2 attacks are the easiest for hackers to deploy. The simplest first step is to completely disable all unused ports on a switch using the shutdown command (Tip: Use interface range to shut down multiple ports at once!).

Mitigate MAC Address Table Attacks

To prevent MAC address table overflow attacks, we use Port Security. Port security limits the number of valid MAC addresses allowed on a port.

  • Enable it using: switchport port-security (Note: The port MUST be set to access mode first).
  • Set max addresses: switchport port-security maximum [value].

The switch can learn MAC addresses in 3 ways: Manually configured, Dynamically learned, or Dynamically Learned - Sticky (which saves the learned MAC address to the running-config automatically).

Port Security Violation Modes & Err-Disabled

If an unauthorized device plugs into the port, a violation occurs. You can choose 1 of 3 violation modes:

  • Shutdown (Default): The port immediately shuts down and goes into an err-disabled state. It sends a log message.
  • Restrict: Drops unauthorized packets, keeps the port up, but sends a log message and increments the violation counter.
  • Protect: The least secure. Drops packets, keeps the port up, but sends NO log message.

To fix an err-disabled port, the admin must manually type shutdown and then no shutdown on the interface.

11.2 Mitigate VLAN Attacks

Preventing VLAN Hopping

A VLAN hopping attack allows a hacker to see traffic on VLANs they shouldn't have access to. Here is the 5-step mitigation checklist:

  1. Disable DTP auto-trunking on user ports: switchport mode access
  2. Disable unused ports and assign them to a "black hole" unused VLAN.
  3. Manually enable trunking on router/switch links: switchport mode trunk
  4. Disable DTP negotiations on trunks: switchport nonegotiate
  5. Change the Native VLAN from VLAN 1 to an unused number to stop double-tagging attacks.

11.3 Mitigate DHCP Attacks

DHCP Snooping

Hackers use tools like Gobbler for DHCP Starvation (stealing all IPs) and DHCP Spoofing (setting up a rogue server). We stop this using DHCP Snooping.

DHCP Snooping creates a table that binds MAC addresses to IP addresses. It forces you to declare which ports are "Trusted" (where the real DHCP server is) and limits how many DHCP requests untrusted user ports can send per second.

11.4 Mitigate ARP Attacks

Dynamic ARP Inspection (DAI)

To prevent ARP Spoofing, a switch must ensure only valid ARP requests/replies are sent. DAI intercepts all ARP traffic on untrusted ports and compares it to the DHCP Snooping binding table.

If a hacker sends an ARP reply claiming to be the router, DAI looks at the table, sees the hacker is lying about their IP-to-MAC binding, and drops the packet!

11.5 Mitigate STP Attacks

PortFast and BPDU Guard

Hackers can manipulate STP by spoofing the Root Bridge to intercept all traffic. We protect the network using two tools on all end-user access ports:

  • PortFast: Bypasses STP's slow listening/learning states so the PC gets a connection immediately.
  • BPDU Guard: Normal PCs should never send BPDU messages. If BPDU Guard sees a BPDU on a user port, it immediately puts the port into an err-disabled state!
