Practice Exam: Communicating Between Networks

1. Which information is used by routers to forward a data packet toward its destination?
Correct Answer: destination IP address

Deep Dive: Routers operate at Layer 3 of the OSI model. Their primary job is to inspect the incoming packet, look at the destination IP address, and compare it against their routing table to determine the best path forward.

  • A. source IP address: (Incorrect) Routers don't use the sender's IP to determine the forward path.
  • B. destination IP address: (Correct) The exact field used to look up routes in the routing table.
  • C & D. source/destination data-link address: (Incorrect) MAC addresses (data-link) are used by Layer 2 switches, not routers.
2. A computer has to send a packet to a destination host in the same LAN. How will the packet be sent?
Correct Answer: Directly to the destination host.

Deep Dive: If two computers are on the exact same subnet (LAN), they do not need a router to talk. The sending PC will use ARP to find the destination MAC address and send the frame directly through the local switch.

  • A & C. To the default gateway: (Incorrect) The default gateway (router) is only used when data needs to leave the local network to reach a remote network (like the Internet).
  • B. Directly to the destination host: (Correct) Local traffic stays local via Layer 2 switching.
3. A router receives a packet from Gigabit 0/0 and determines it needs to be forwarded out Gigabit 0/1. What will the router do next?
Correct Answer: Create a new Layer 2 Ethernet frame to be sent to the destination

Deep Dive: A packet cannot travel over physical cables naked; it must be encapsulated inside a Layer 2 frame. When a router receives a frame, it strips off the old Layer 2 header, looks at the Layer 3 IP packet to find the route, and then must build a brand new Layer 2 frame to push it out the exit interface.

  • A. Route directly: (Incorrect) It cannot send raw packets; encapsulation is required.
  • B. Create a new Layer 2 Ethernet frame: (Correct) The router encapsulates the packet with a new Source MAC (itself) and Destination MAC (the next hop).
  • C. Look into the ARP cache: (Incorrect) It does this *during* frame creation to find the destination MAC.
  • D. Look into the routing table: (Incorrect) It already did this to determine that G0/1 was the exit interface.
4. Which IPv4 address can a host use to ping the loopback interface?
Correct Answer: 127.0.0.1

Deep Dive: The IPv4 address block 127.0.0.0/8 is reserved for loopback testing. Pinging your own loopback address tests if your computer's internal TCP/IP software stack is installed and functioning correctly.

  • A. 126.0.0.1: (Incorrect) This is a standard Class A public IP address.
  • B. 127.0.0.0: (Incorrect) This is the network address for the entire loopback block.
  • C. 127.0.0.1: (Correct) The universally standard loopback testing IP address.
5. A computer can access devices on the same network but cannot access devices on other networks. What is the probable cause?
Correct Answer: The computer has an invalid default gateway address.

Deep Dive: If you can talk to local devices (like a network printer), your IP, subnet, and cables are working fine. However, to leave your local network and reach the internet, your PC must forward traffic to the router. If your default gateway IP is wrong, your PC is trapped locally.

  • A. Cable not connected: (Incorrect) You wouldn't be able to access local devices either.
  • B & C. Invalid IP / Subnet: (Incorrect) This would break local communication.
  • D. Invalid default gateway: (Correct) The gateway is the only door out of the local LAN.
6. Which statement describes a feature of the IP protocol?
Correct Answer: IP relies on upper layer services to handle missing or out-of-order packets.

Deep Dive: IP is a "best-effort", connectionless protocol. It sends packets into the void and hopes they arrive. It has no built-in tracking for lost packets. It relies entirely on Layer 4 protocols (like TCP) to acknowledge receipt and ask for retransmissions if something is missing.

  • A. Modified based on media: (Incorrect) This describes Layer 2 frames, not Layer 3 IP packets.
  • B. Relies on L2 for error control: (Incorrect) It relies on Layer 4 (TCP) for reliable delivery.
  • C. MAC addresses used: (Incorrect) IP uses IP addresses. Layer 2 uses MAC addresses.
  • D. Relies on upper layer services: (Correct) TCP handles the reliability that IP lacks.
7. Why is NAT not needed in IPv6?
Correct Answer: Any host can get a public IPv6 address because the available number is extremely large.

Deep Dive: Network Address Translation (NAT) was created as a band-aid because we ran out of the 4.3 billion IPv4 addresses. IPv6 has 340 undecillion addresses (enough for every atom on Earth to have an IP). Because addresses are essentially infinite, every device can have its own public IP, making NAT obsolete.

  • A. IPv6 has integrated security: (Incorrect) While IPsec is built-in, NAT was created for address conservation, not security.
  • B. Extremely large number available: (Correct) Eradicates address exhaustion.
  • C. Solved by routers: (Incorrect) NAT isn't solved by routers, it's just no longer necessary.
8. Which parameter does the router use to choose the path to the destination when multiple routes are available?
Correct Answer: the lower metric value associated with the destination network

Deep Dive: In routing, a "Metric" represents the cost or distance of a path (e.g., hop count in RIP, or bandwidth cost in OSPF). Just like driving, a router always wants the shortest, cheapest, fastest path. Therefore, it always chooses the route with the lowest metric.

  • A. lower metric value: (Correct) Lower cost means a better path.
  • B. lower gateway IP: (Incorrect) IP numbers have no bearing on path quality.
  • C. higher metric value: (Incorrect) Higher metrics indicate slower/worse paths.
9. What are two services provided by the OSI network layer? (Combined)
Correct Answer: routing packets AND encapsulating PDUs from the transport layer

Deep Dive: The Network Layer (Layer 3) takes segments from the Transport layer, encapsulates them into IP Packets, and uses logical addressing (IP addresses) to route those packets across multiple networks to their final destination.

  • A & C. Error detection / media placement: (Incorrect) These are the primary jobs of the Data Link Layer (Layer 2).
  • B. routing packets AND encapsulating PDUs: (Correct) The exact functions of Layer 3 IP.
10. Within a production network, what is the purpose of configuring a switch with a default gateway address?
Correct Answer: It is used to forward packets originating from the switch to remote networks.

Deep Dive: A Layer 2 switch doesn't need an IP address or a gateway to forward local traffic. However, if a network admin is sitting at home and wants to remotely SSH into the switch, the switch needs a default gateway so it knows how to send the return SSH packets back out to the internet.

  • A. Hosts use it: (Incorrect) Hosts use the router's IP as their gateway, not the switch's IP.
  • B. Accessible locally: (Incorrect) Local management only requires an IP on the same subnet, no gateway needed.
  • C. Forward packets originating from switch: (Correct) Required for remote management traffic.
11. What is a basic characteristic of the IP protocol?
Correct Answer: connectionless

Deep Dive: IP (Internet Protocol) is connectionless. It acts like sending a postcard through the mail. You drop it in the mailbox without checking to see if the recipient is home, and you get no confirmation receipt if it gets lost. (TCP handles the reliable connection portion).

  • A. connectionless: (Correct) IP sends packets immediately without establishing a session first.
  • B. media dependent: (Incorrect) IP is media independent (it runs over copper, fiber optic, and Wi-Fi equally well).
  • C. reliable delivery: (Incorrect) IP is unreliable. TCP is reliable.
12. Which field in the IPv4 header is used to prevent a packet from traversing a network endlessly?
Correct Answer: Time-to-Live

Deep Dive: If routers get confused, they could bounce a packet back and forth forever, crashing the internet. The Time-to-Live (TTL) field starts at a number (like 128). Every time a router touches the packet, it subtracts 1. If the TTL hits 0, the router throws the packet in the trash, ending the loop.

  • A. Time-to-Live: (Correct) The countdown timer preventing infinite routing loops.
  • B. Sequence Number: (Incorrect) A TCP field used to reorder packets.
  • C. Differentiated Services: (Incorrect) DSCP is used to prioritize traffic (QoS).
13. What is one advantage that the IPv6 simplified header offers over IPv4?
Correct Answer: efficient packet handling

Deep Dive: The IPv4 header had a lot of unnecessary fluff, like the Header Checksum, which forced routers to pause and do math on every single packet. IPv6 stripped all that fluff out. Because the header is perfectly streamlined, routers can forward IPv6 packets incredibly fast.

  • A & B. Smaller header / addresses: (Incorrect) IPv6 addresses are massive (128-bit vs 32-bit), so the header is actually much larger in byte size (40 bytes vs 20 bytes).
  • C. efficient packet handling: (Correct) The streamlined design means less processing overhead for routers.
14. What IPv4 header field identifies the upper layer protocol carried in the packet?
Correct Answer: Protocol

Deep Dive: When a router opens an IP packet, it needs to know what is hidden inside the payload. The 8-bit Protocol field acts as a label. (e.g., Code 6 means "There is a TCP segment inside", Code 17 means "UDP", and Code 1 means "ICMP").

  • A. Protocol: (Correct) Identifies the next-level payload type.
  • B. Identification: (Incorrect) Used to reassemble fragmented packets.
  • C. Version: (Incorrect) Just identifies if it is IPv4 or IPv6.
15. In routing, if a packet's destination network has NO exact entry in the routing table, what will the router do?
Correct Answer: Send it to the Gateway of Last Resort (if configured).

Deep Dive: A router checks its routing table from most specific to least specific. If the destination IP is nowhere in the table, the router uses its Default Route (0.0.0.0/0), also known as the Gateway of Last Resort, to forward the packet toward the internet. If no default route exists, THEN it drops the packet.

  • A. Drop immediately: (Incorrect) It checks for a default route first.
  • B. Flood the packet: (Incorrect) Routers NEVER flood unknown unicast packets; only switches do that.
  • C. Gateway of Last Resort: (Correct) The catch-all default route.
16. What information does the loopback test provide?
Correct Answer: The TCP/IP stack on the device is working correctly.

Deep Dive: Pinging 127.0.0.1 sends a packet straight down into your PC's Network Interface Card (NIC) and bounces it right back up. It verifies that your computer's internal networking software (TCP/IP) is installed and functioning properly.

  • A. TCP/IP stack working: (Correct) Tests internal host software.
  • B. End-to-end connectivity: (Incorrect) You must ping a remote server to test this.
  • C. Ethernet cable working: (Incorrect) A loopback test succeeds even if the cable is unplugged.
17. What routing table entry has a next hop address associated with a destination network?
Correct Answer: remote routes

Deep Dive: If a network is "directly connected" (like plugging a switch straight into the router), the router knows exactly how to reach it. But if a network is "remote" (several miles away), the router doesn't know where it is. It only knows the IP address of the neighboring router it must pass the packet to. This neighboring router is called the Next Hop.

  • A. directly-connected: (Incorrect) Reached via local interfaces, not next hops.
  • B. remote routes: (Correct) Requires forwarding to a next-hop IP address.
  • C. local routes: (Incorrect) Represents the exact IP on the router's own interface.
18. How do hosts ensure that their packets are directed to the correct network destination?
Correct Answer: They search in their own local routing table for a route and pass it to the default gateway.

Deep Dive: Did you know Windows PCs have routing tables too? Before your computer sends a packet, it checks its internal Host Routing Table to see if the destination IP is on the same local subnet. If it is local, it sends it directly to the PC. If the destination is remote, the table tells the PC to forward it to the Default Gateway.

  • A. Always to default gateway: (Incorrect) Local traffic skips the gateway entirely.
  • B. Search own local routing table: (Correct) PC operating systems make routing decisions.
  • C. Send query to gateway: (Incorrect) This is not how standard IP functions.
19. When transporting data from real-time applications (streaming audio/video), which IPv6 field informs routers to maintain the same path?
Correct Answer: Flow Label

Deep Dive: If video packets take different paths through the internet, some will arrive fast and others will arrive late, causing a glitchy, out-of-order video stream. The IPv6 Flow Label field tags all the packets in a video stream with the same ID, instructing routers to keep them all on the exact same physical path to prevent stuttering.

  • A. Next Header: (Incorrect) Identifies the payload protocol (like TCP/UDP).
  • B. Flow Label: (Correct) A 20-bit field preventing packet re-ordering for real-time streams.
  • C. Traffic Class: (Incorrect) Used for general QoS packet prioritization.
20. What statement describes the function of the Address Resolution Protocol (ARP)?
Correct Answer: It discovers the MAC address of any host on the local network.

Deep Dive: To send data, a PC needs two things: the IP address and the MAC address. If you type an IP address, your PC shouts an ARP Broadcast saying, "Hey, whoever owns this IP address, reply and tell me your MAC address!"

  • A. Discovers IP on different network: (Incorrect) DNS handles IP discovery.
  • B. Discovers MAC on local network: (Correct) Maps known IPv4 addresses to unknown MAC addresses locally.
  • C. Discovers MAC on different network: (Incorrect) MAC addresses never cross routers; ARP only works locally.
21. Under which two circumstances will a switch flood a frame out of every port except the incoming port? (Combined)
Correct Answer: The frame has the broadcast address AS WELL AS when the destination address is unknown.

Deep Dive: A switch forwards frames specifically to the destination MAC address. However, if the destination MAC is FF:FF:FF:FF:FF:FF (Broadcast), the switch knows it MUST copy it to everyone. Additionally, if the destination MAC is not in the switch's MAC Table yet (Unknown Unicast), it panics and floods the frame everywhere to ensure it reaches the target.

  • A. Broadcast AND Unknown destination: (Correct) The two triggers for switch flooding.
  • B & C. Source is broadcast/multicast: (Incorrect) A source MAC address is NEVER a broadcast or multicast address. It is always a single unicast sender.
22. Which statement describes the treatment of ARP requests on the local link?
Correct Answer: They are received and processed by every device on the local network.

Deep Dive: Because ARP requests do not know the target's MAC address, they are sent as a Layer 2 Broadcast. This forces every single Network Interface Card (NIC) on the subnet to receive the frame, open it, and process the payload to see if they are the intended IP address.

  • A. Forwarded by routers: (Incorrect) Routers drop Layer 2 broadcasts; they do not forward them.
  • B. Processed by every device: (Correct) The inherent behavior of a broadcast frame.
  • C. Dropped by switches: (Incorrect) Switches flood broadcasts, they don't drop them.
23. Which destination address is used in an ARP request frame?
Correct Answer: FFFF.FFFF.FFFF

Deep Dive: An ARP request is a Layer 2 function used to find a MAC address. Because it must reach everyone on the local switch, the Destination MAC address placed on the Ethernet header is the universal Layer 2 Broadcast Address, which is all Fs in hexadecimal.

  • A. 0.0.0.0: (Incorrect) Unspecified IPv4 address.
  • B. 255.255.255.255: (Incorrect) This is the Layer 3 broadcast IP address, but the question specifies the *frame* (Layer 2).
  • C. FFFF.FFFF.FFFF: (Correct) The Layer 2 Broadcast MAC Address.
24. A network technician issues the arp -d * command on a PC. What is the result?
Correct Answer: The ARP cache is cleared.

Deep Dive: On a Windows PC, the arp command manages the ARP table. Typing arp -a displays the current cache. Typing arp -d * tells the system to delete (*) all entries. This flushes the cache, forcing the PC to send fresh ARP requests and relearn network MAC addresses.

25. PC1 sends a frame addressed to PC3. However, PC3's MAC address is NOT in the switch's MAC address table. What will the switch do?
Correct Answer: It will forward the frame to all ports except the incoming port.

Deep Dive: This scenario is called Unknown Unicast Flooding. The switch doesn't know what port PC3 is plugged into yet. Instead of dropping the frame, it plays it safe and blasts copies of the frame out of every active port. It excludes the port PC1 is on because it violates the Split-Horizon rule (never send data back out the port it arrived on).

26. Which two types of IPv6 messages are used in place of ARP for address resolution?
Correct Answer: neighbor solicitation & neighbor advertisement

Deep Dive: IPv6 completely eliminated ARP broadcasts. Instead, IPv6 uses the ICMPv6 Neighbor Discovery Protocol (NDP). To find a MAC address, an IPv6 host sends a Neighbor Solicitation (NS) message, and the target replies with a Neighbor Advertisement (NA) message.

  • A. Echo reply/request: (Incorrect) These are standard Ping messages.
  • B. Solicitation & Advertisement: (Correct) The direct IPv6 replacement for ARP Request/Reply.
  • C. anycast & broadcast: (Incorrect) IPv6 doesn't even use broadcasts.
27. What is the aim of an ARP spoofing attack?
Correct Answer: to associate IP addresses to the wrong MAC address

Deep Dive: ARP has no security; it blindly trusts any reply it receives. In an ARP Spoofing (or ARP Poisoning) attack, a hacker sends a fake reply to a victim PC saying, "I am the Router!" The victim PC updates its ARP cache, linking the Router's IP to the Hacker's MAC address. Now, all internet traffic routes through the hacker (Man-in-the-Middle).

  • A. associate IP to wrong MAC: (Correct) The exact mechanism of ARP poisoning.
  • C. fill switch MAC tables: (Incorrect) This describes MAC Flooding, not ARP Spoofing.
28. PC1 connects to a remote File Server and sends an ARP request. Which MAC address will PC1 receive in the ARP reply?
Correct Answer: the MAC address of the default gateway router

Deep Dive: ARP broadcasts cannot cross routers. If PC1 wants to talk to a remote server, it realizes the IP is on a different subnet. Therefore, PC1 must send the frame to its router. To do this, PC1 sends an ARP request asking for the MAC address of its Default Gateway, NOT the server.

  • A. Default gateway MAC: (Correct) Used as the Layer 2 destination to escape the local LAN.
  • B. Switch MAC: (Incorrect) Switches are transparent; PCs don't address traffic to them.
  • C. File Server MAC: (Incorrect) The server's MAC is unknown and unreachable via local ARP.
29. Where are IPv4 address to Layer 2 Ethernet address mappings maintained on a host computer?
Correct Answer: ARP cache

Deep Dive: Every computer keeps a temporary list of IP addresses and their matching MAC addresses to avoid sending ARP broadcasts every time it needs to communicate. This mapping table is called the ARP cache.

  • A. ARP cache: (Correct) The IPv4 to MAC mapping table.
  • B. neighbor table: (Incorrect) This is the term used for IPv6 mapping.
  • C. routing table: (Incorrect) Maps destination IPs to next-hop routes.
30. What important information is examined in the Ethernet frame header by a Layer 2 device in order to forward the data onward?
Correct Answer: destination MAC address

Deep Dive: A Layer 2 switch makes forwarding decisions based exclusively on the Destination MAC address. It looks at the destination MAC on the frame, checks its internal MAC Address Table, and forwards the frame out the specific port associated with that MAC.

  • A. source MAC address: (Incorrect) The source MAC is examined to *build* the table, not to forward the frame.
  • B. destination MAC address: (Correct) Used to determine the exit port.
  • C. destination IP address: (Incorrect) This is a Layer 3 function used by routers.
31. Which command is used to configure a message that displays after accessing the router?
Correct Answer: banner motd #

Deep Dive: For legal and security reasons, companies configure a "Message of the Day" (MOTD) banner that warns unauthorized users that access is restricted. This message displays instantly upon connecting to the terminal, before the login prompt appears.

  • A. password class: (Incorrect) Configures the enable password.
  • B. banner motd #: (Correct) The command to set the warning banner.
  • C. hostname R1: (Incorrect) Changes the device name.
32. What is the fastest way a network administrator could test whether the banner is properly configured?
Correct Answer: Exit privileged EXEC mode and press Enter.

Deep Dive: The MOTD banner displays immediately when a session connects. By typing exit until you completely log out of the router, you simulate a fresh connection. Hitting 'Enter' re-initiates the console session, instantly flashing the banner on your screen.

  • A & C. Reboot / Power cycle: (Incorrect) This takes several minutes and brings down the entire network unnecessarily.
  • B. Exit and press Enter: (Correct) The fastest, non-disruptive way to view the login screen.
33. Which remote access method relies on an active network connection and uses encryption?
Correct Answer: SSH

Deep Dive: Secure Shell (SSH) is the industry standard for logging into routers and switches remotely. Because it uses strong cryptography (RSA keys), all passwords and commands sent over the network are completely encrypted and hidden from packet sniffers.

  • A. SSH: (Correct) Remote, active network connection WITH encryption.
  • B. Telnet: (Incorrect) Remote connection but completely unencrypted (plaintext).
  • C. Console: (Incorrect) Requires a physical blue rollover cable, not an active IP network connection.
34. What is the correct sequence of the three major phases of a Cisco router bootup process?
Correct Answer: POST -> Load IOS -> Load startup-config

Deep Dive: A router boots in three logical steps:
1. POST (Power-On Self Test): Checks hardware like CPU and RAM.
2. Load IOS: Finds and loads the operating system from Flash memory.
3. Load startup-config: Applies the saved settings from NVRAM to the active running-config.

35. In which device mode is the service password-encryption command entered?
Correct Answer: R1(config)#

Deep Dive: The service password-encryption command automatically scrambles all plaintext passwords in your configuration file. Because this is a system-wide setting affecting the entire device, it must be entered in Global Configuration Mode.

  • A. R1#: (Incorrect) Privileged EXEC mode is for viewing (show commands), not configuring.
  • B. R1(config)#: (Correct) Global Configuration mode.
  • C. R1(config-line)#: (Incorrect) Line config mode is for specific ports (like vty or console).
36. Which two functions are properties of NVRAM? (Combined)
Correct Answer: It retains contents when power is removed AND stores the startup config.

Deep Dive: NVRAM stands for Non-Volatile RAM. "Non-Volatile" means the memory does not wipe when the power cord is pulled. Its sole purpose on a Cisco router is to permanently store the startup-config file.

  • A. Retains contents / stores startup config: (Correct) The exact definition of NVRAM.
  • B & C. Routing table / ARP table / running config: (Incorrect) These are highly active processes stored in standard Volatile RAM, which wipes clean when rebooted.
37. A router boots and enters setup mode. What is the reason for this?
Correct Answer: The configuration file is missing from NVRAM.

Deep Dive: When a Cisco router boots up, it looks inside NVRAM for the startup-config file. If the NVRAM is empty, the router assumes it is brand new out of the box and automatically enters the interactive "Setup Mode" dialog to help you configure it.

  • A. IOS missing: (Incorrect) If the OS is missing, it boots into ROMMON mode.
  • B. POST hardware failure: (Incorrect) The router would likely halt or crash.
  • C. Config missing from NVRAM: (Correct) Triggers the initial Setup Mode.
38. The global configuration command ip default-gateway 172.16.100.1 is applied to a switch. What is the effect?
Correct Answer: The switch can be remotely managed from a host on another network.

Deep Dive: A Layer 2 switch does not route traffic, so it doesn't strictly *need* a default gateway. However, if a network admin tries to SSH into the switch from their house across the internet, the switch needs a default gateway so it knows where to send the return SSH packets.

  • A. Remotely managed from another network: (Correct) The only reason a Layer 2 switch needs a gateway.
  • B. Communicate with hosts on 172.16.100.0: (Incorrect) It doesn't need a gateway for local communication.
  • C. Management interface address: (Incorrect) The command sets the gateway, not the switch's own IP address.
39. What happens when the transport input ssh command is entered on the switch vty lines?
Correct Answer: Communication between the switch and remote users is encrypted.

Deep Dive: By default, Cisco VTY lines accept all remote connections, including highly insecure Telnet (which transmits passwords in plaintext). Applying transport input ssh explicitly blocks Telnet and forces all remote administrators to use SSH, guaranteeing that all communication is encrypted.

  • A. SSH client enabled: (Incorrect) This command configures the SSH *Server* on the switch, not the client.
  • B. Communication is encrypted: (Correct) SSH forces strong encryption.
  • C. Proprietary software: (Incorrect) SSH is an open industry standard.
40. A user PC has successfully transmitted packets to www.cisco.com. Which IP address does the user PC target in order to forward its data off the local network?
Correct Answer: The Default Gateway IP

Deep Dive: When a PC needs to send a packet to an IP address that is not on its local subnet (like a website on the Internet), it must hand the packet to a router. To do this, the PC forwards the Layer 2 Ethernet frame to the MAC address of its Default Gateway.

  • A. DNS Server IP: (Incorrect) DNS is only queried to translate the URL into an IP address.
  • B. Web Server IP: (Incorrect) This is the final Layer 3 destination, but not the local target used to *leave* the network.
  • C. Default Gateway IP: (Correct) The exit door of the local network.
41. Which prompt represents Privileged EXEC mode where you can save the configuration?
Correct Answer: R1#

Deep Dive: Cisco IOS uses different command-line prompts. The > symbol is User EXEC mode (very limited viewing). The # symbol is Privileged EXEC mode, which gives you full access to view configurations and execute the copy running-config startup-config command to save your work.

42. Which three commands are used to set up secure access to a router through a connection to the console interface?
Correct Answer: line console 0, password cisco, login

Deep Dive: To secure the physical blue console port on a router, you must first enter the line configuration mode for it (line console 0). Then, set the password (password cisco), and finally, tell the router to actually prompt users for it (login).

  • A. line vty: (Incorrect) This configures remote network access (SSH/Telnet), not the physical console.
  • B. enable secret: (Incorrect) This command is applied globally, not inside the console line.
  • C. Correct sequence: (Correct) Accesses the console line, applies the password, and enforces it.
43. What is a description of the default gateway address?
Correct Answer: It is the IP address of the Router interface that connects the PC LAN to the Router.

Deep Dive: The default gateway is the local doorway out of your network. It must be an IP address on the exact same subnet as your PC, meaning it is the physical router interface directly facing the internal LAN.

  • A. Router interface connecting LAN: (Correct) The local entry point into the router.
  • B. Interface connecting Internet: (Incorrect) This is the router's WAN interface (Public IP). PCs don't use this as their gateway.
  • C. Switch IP: (Incorrect) Switches are Layer 2 and do not act as gateways for PCs.
44. Which two functions are primary functions of a router? (Combined)
Correct Answer: packet forwarding AND path selection

Deep Dive: A router only has two main jobs. 1) Path Selection: Looking at the routing table to decide the best route to the destination. 2) Packet Forwarding: Moving the packet from the incoming interface to the correct outgoing interface.

45. What is the effect of using the copy running-config startup-config command on a router?
Correct Answer: The contents of NVRAM will change.

Deep Dive: The running configuration is stored in volatile RAM (which erases on reboot). The startup-config is stored in NVRAM (Non-Volatile RAM, which survives reboots). When you copy the running config to the startup config, you are overwriting the contents of NVRAM to permanently save your work.

46. What will happen if the default gateway address is incorrectly configured on a host?
Correct Answer: The host cannot communicate with hosts in other networks.

Deep Dive: The default gateway is ONLY used when a PC realizes the destination IP is on a remote subnet. If the gateway is missing or wrong, the PC drops the packet because it has no exit door. However, local communication (like talking to a nearby printer) bypasses the gateway entirely and will continue to work perfectly.

47. What are two potential network problems that can result from ARP operation? (Combined)
Correct Answer: Multiple ARP broadcasts cause delays AND attackers can manipulate mappings to intercept traffic.

Deep Dive: ARP has two massive flaws. 1) Performance: It relies on Layer 2 Broadcasts, which interrupt every single PC on the network and cause delays. 2) Security: It has zero authentication, allowing hackers to execute ARP Spoofing (ARP Poisoning) to easily execute Man-in-the-Middle attacks.

48. Packet Tracer Verification: Using show ip interface brief, which condition proves an interface is active and operational?
Correct Answer: It shows "up" in both the Status and Protocol columns.

Deep Dive: An interface must be "Up" (Layer 1 Physical connectivity) and "Up" (Layer 2 Data-link protocol operational) to successfully route traffic.

49. Which term describes a field in the IPv4 packet header used to identify the next level protocol?
Correct Answer: Protocol

Deep Dive: The 8-bit Protocol field inside the IPv4 header acts as a label telling the receiving computer what payload is inside (e.g., Code 6 for TCP, Code 17 for UDP, Code 1 for ICMP).

50. Which term describes a field in the IPv4 packet header that contains an 8-bit binary value used to determine the priority of each packet?
Correct Answer: Differentiated Services (DS)

Deep Dive: The Differentiated Services field (formerly Type of Service) contains the DSCP value. This is used by QoS engines to prioritize sensitive traffic like Voice over IP (VoIP) ahead of standard file downloads.

51. Which term describes a field in the IPv4 packet header that contains a 32-bit binary value associated with an interface on the sending device?
Correct Answer: Source IPv4 Address

Deep Dive: Every IPv4 header has a 32-bit (4 byte) Source IP Address field, which identifies the original device that generated the packet.

52. Which term describes a field in the IPv4 packet header used to detect corruption in the IPv4 header?
Correct Answer: Header Checksum

Deep Dive: The Header Checksum is a mathematical calculation used to verify that the IPv4 header was not corrupted by electrical noise during transmission. (Note: IPv6 completely eliminated this field to speed up routing!).

54. Which term describes a field in the IPv4 packet header that contains a unicast, multicast, or broadcast address?
Correct Answer: Destination IPv4 Address

Deep Dive: The Destination IP Address defines where the packet is going. It can be aimed at a single PC (Unicast), a group of PCs (Multicast), or every PC on the subnet (Broadcast).

55. Which term describes a field in the IPv4 packet header used to limit the lifetime of a packet?
Correct Answer: TTL

Deep Dive: The Time-to-Live (TTL) field decreases by 1 every time a router processes the packet. If the TTL hits 0, the packet dies, preventing it from looping around the internet forever.

56. Which term describes a field in the IPv4 packet header that contains a 4-bit binary value set to 0100?
Correct Answer: Version

Deep Dive: The very first 4 bits of the IP header indicate the IP Version. The binary value 0100 equals the decimal number 4 (indicating IPv4). If it were an IPv6 packet, the binary would be 0110 (6).

57. What property of ARP causes cached IP-to-MAC mappings to remain in memory longer?
Correct Answer: Entries in an ARP table are time-stamped and are purged after the timeout expires.

Deep Dive: ARP entries are dynamic. When a PC learns a MAC address, it stamps it with a timer (usually a few minutes). As long as the timer hasn't expired, the PC remembers the MAC address and avoids sending another ARP Broadcast.

58. What property of ARP allows MAC addresses of frequently used servers to be fixed in the ARP table?
Correct Answer: A static IP-to-MAC address entry can be entered manually into an ARP table.

Deep Dive: If you have a critical Server, you can manually type its MAC address into the router or PC's ARP table. Static entries do not have a timeout timer, and they are immune to ARP Spoofing because the router will never overwrite a manual entry with a dynamic fake one.

59. What property of ARP allows hosts on a LAN to send traffic to remote networks?
Correct Answer: Local hosts learn the MAC address of the default gateway.

Deep Dive: ARP broadcasts cannot cross routers. To reach the Internet, a PC sends an ARP request asking for the MAC address of its Default Gateway. It then encapsulates the remote IP packet inside a Layer 2 frame addressed to the router's MAC.

61. What property of ARP forces all Ethernet NICs to process an ARP request?
Correct Answer: The destination MAC address FF-FF-FF-FF-FF-FF appears in the header.

Deep Dive: Network Interface Cards (NICs) generally ignore frames that aren't addressed to their own specific MAC address. However, FF-FF-FF-FF-FF-FF is the universal Layer 2 Broadcast address. By design, every NIC is hardcoded to accept and process frames sent to this address.

62. What property of ARP causes a reply only to the source sending an ARP request?
Correct Answer: The source MAC address appears in the header of the Ethernet frame.

Deep Dive: When a PC sends an ARP Request broadcast, it includes its own Source MAC address. When the target device builds its ARP Reply, it copies that original Source MAC and uses it as the new Destination MAC. This allows the reply to be sent as a direct Unicast, rather than another noisy broadcast.

63. What property of ARP causes the request to be flooded out all ports of a switch except for the port receiving the ARP request?
Correct Answer: The destination MAC address FF-FF-FF-FF-FF-FF appears in the header.

Deep Dive: Switches are programmed to flood Layer 2 Broadcasts (FF-FF-FF-FF-FF-FF) out of all active ports. They intentionally exclude the port it arrived on to comply with the Split-Horizon rule, preventing physical loops.

64. What property of ARP causes the NICs receiving an ARP request to pass the data portion of the Ethernet frame to the ARP process?
Correct Answer: The type field 0x806 appears in the header of the Ethernet frame.

Deep Dive: The "Type" field in a Layer 2 Ethernet header tells the computer what protocol is hiding inside the payload. 0x0800 means IPv4. 0x86DD means IPv6. 0x0806 is the specific hex code that tells the computer: "This payload contains an ARP request."
