Module 8 Quiz: SLAAC & DHCPv6

1. How does an IPv6 client ensure that it has a unique address after it configures its IPv6 address using the SLAAC allocation method?
Correct Answer: It sends an ICMPv6 Neighbor Solicitation message...

Deep Dive: Because SLAAC is stateless (meaning no server tracks who has what IP), the host generates its own IPv6 address. To make absolutely sure no one else on the network accidentally generated the exact same IP, the host performs DAD (Duplicate Address Detection). It does this by broadcasting an ICMPv6 Neighbor Solicitation message asking, "Is anyone already using this IP?"

  • A & C. Database / DHCPv6 Server: (Incorrect) SLAAC does not use a central database or server.
  • B. Sends ICMPv6 Neighbor Solicitation: (Correct) This is the mechanism used to perform DAD.
  • D. ARP message: (Incorrect) ARP is strictly used in IPv4. IPv6 uses ICMPv6 Neighbor Discovery instead.
2. Which method would an IPv6-enabled host using SLAAC employ to learn the address of the default gateway?
Correct Answer: Router advertisements that are received from the link router

Deep Dive: In an IPv6 network, the router periodically broadcasts Router Advertisement (RA) messages. The source address of this RA message is the router's own Link-Local Address (FE80::). When a host running SLAAC receives this RA, it automatically sets that link-local address as its default gateway.

  • A. Router advertisements: (Correct) RAs contain the network prefix and the router's link-local address.
  • B & D. DHCPv6 server messages: (Incorrect) SLAAC specifically avoids relying on a DHCPv6 server for basic IP and gateway info.
  • C. Neighbor advertisements: (Incorrect) NAs are used to map MAC addresses to IP addresses, not to hand out default gateways.
3. What two methods can be used to generate an interface ID by an IPv6 host that is using SLAAC? (Combined)
Correct Answer: Random generation AND EUI-64

Deep Dive: When using SLAAC, the router provides the first 64 bits (the Network Prefix). The host must make up the last 64 bits (the Interface ID) by itself. It does this in one of two ways: 1) Using EUI-64, which takes the device's MAC Address, splits it in half, and inserts `FFFE` in the middle, or 2) Using a purely random 64-bit number (which modern OSs like Windows 10 do for privacy).

  • A. Stateful DHCPv6: (Incorrect) This is the opposite of SLAAC.
  • B. Random generation AND EUI-64: (Correct) The two standard methods an OS uses to invent its own IPv6 Interface ID.
  • C & D. ARP / DAD: (Incorrect) ARP is IPv4 only. DAD is used to check if an address is unique, not to generate it.
4. A client is using SLAAC to obtain an IPv6 address for its interface. After an address has been generated and applied to the interface, what must the client do before it can begin to use this IPv6 address?
Correct Answer: It must send an ICMPv6 Neighbor Solicitation message to ensure that the address is not already in use on the network.

Deep Dive: Because SLAAC allows devices to essentially invent their own IP addresses without asking a server, there is a tiny chance two devices might generate the exact same IP. Before routing any real data, the host must perform Duplicate Address Detection (DAD) using an ICMPv6 Neighbor Solicitation message to ask the network if the IP is taken.

  • A. Send Neighbor Solicitation: (Correct) This performs DAD to guarantee uniqueness.
  • B. Send DHCPv6 REQUEST: (Incorrect) SLAAC does not ask a DHCP server for permission to use an IP.
  • C. Send INFORMATION-REQUEST: (Incorrect) While Stateless DHCPv6 does this to get DNS, it is not required just to *use* the IP address itself.
  • D. Send Router Solicitation: (Incorrect) RS messages are sent *before* the IP is generated to find the router prefix, not after.
5. Which command should be configured on a router interface to set the router as a stateful DHCPv6 client?
Correct Answer: ipv6 address dhcp

Deep Dive: Sometimes a router itself needs to obtain an IPv6 address from an ISP's upstream DHCPv6 server. To make a router interface act as a client and request a stateful IP address lease, you use the ipv6 address dhcp interface command.

  • A. ipv6 address autoconfigure: (Incorrect) This tells the router interface to act as a client and use SLAAC, not stateful DHCPv6.
  • B. ipv6 dhcp server: (Incorrect) This makes the router act as a server handing OUT addresses, not a client.
  • C. ipv6 enable: (Incorrect) This merely turns on IPv6 processing and generates a link-local address.
  • D. ipv6 address dhcp: (Correct) Configures the interface to acquire an IPv6 address via a stateful DHCPv6 server.
6. What message informs IPv6 enabled interfaces to use stateful DHCPv6 for obtaining an IPv6 address?
Correct Answer: The ICMPv6 Router Advertisement

Deep Dive: When an IPv6 host boots up, it doesn't immediately know how it's supposed to get an IP address. It waits for the router to send an ICMPv6 Router Advertisement (RA). If the router has the M-Flag (Managed) set to 1 inside that RA message, it acts as a direct command telling the host: "Stop using SLAAC. Go find a Stateful DHCPv6 server to get your IP address."

  • A. Router Advertisement: (Correct) The RA message dictates the IP assignment method (SLAAC, Stateless DHCPv6, or Stateful DHCPv6).
  • B. Router Solicitation: (Incorrect) This is sent by the host asking the router to send an RA.
  • C & D. Advertise / Reply: (Incorrect) These are DHCPv6 messages sent *after* the host has already been told to use DHCPv6 by the RA.
7. Which destination IP address is used when an IPv6 host sends a DHCPv6 SOLICIT message to locate a DHCPv6 server?
Correct Answer: FF02::1:2

Deep Dive: In IPv4, DHCP Discover messages are broadcasted to everyone (255.255.255.255). IPv6 does not use broadcasts; it uses highly specific multicast addresses. When a client needs an IP, it sends a SOLICIT message to the All-DHCPv6-Servers-and-Relay-Agents multicast address, which is FF02::1:2.

  • A. FE80::1: (Incorrect) A link-local unicast address.
  • B. FF02::1:2: (Correct) The multicast address exclusively for reaching DHCPv6 servers and relay agents.
  • C. FF02::2: (Incorrect) The multicast address for ALL routers on the local link.
  • D. FF02::1: (Incorrect) The multicast address for ALL IPv6 nodes (devices) on the local link.
8. In which alternative to DHCPv6 does a router dynamically provide IPv6 configuration information to hosts?
Correct Answer: SLAAC

Deep Dive: SLAAC (Stateless Address Autoconfiguration) is the primary alternative to using a DHCPv6 server. The router provides the network prefix, default gateway, and even DNS server info directly in its RA messages, allowing hosts to completely self-configure.

  • A. ICMPv6: (Incorrect) This is the protocol *used* to deliver the messages, not the configuration method itself.
  • B. SLAAC: (Correct) The standard stateless alternative to DHCPv6.
  • C. EUI-64: (Incorrect) A method for creating an interface ID, not a full network configuration delivery method.
  • D. ARP: (Incorrect) Address Resolution Protocol is exclusively used in IPv4.
9. A company implements the stateless DHCPv6 method. After a workstation receives ADVERTISE messages from multiple DHCPv6 servers to indicate their availability, which message does it send to a server to get DNS information?
Correct Answer: DHCPv6 INFORMATION-REQUEST

Deep Dive: In Stateless DHCPv6, the client already created its own IP address using SLAAC. It doesn't need to REQUEST an IP lease. It only needs "extra" information, like the IP address of the DNS server. Therefore, it sends a specific INFORMATION-REQUEST message to the DHCPv6 server.

  • A. INFORMATION-REQUEST: (Correct) Used in Stateless DHCPv6 to ask for DNS/Domain info without asking for an IP address.
  • B. REQUEST: (Incorrect) Used in Stateful DHCPv6 to officially request an IP address lease.
  • C & D. Solicit / Advertise: (Incorrect) These are the initial discovery steps.
10. What process is used in ICMPv6 for a host to verify that an IPv6 address is unique before configuring it on an interface?
Correct Answer: DAD

Deep Dive: Duplicate Address Detection (DAD) is a critical safety check in IPv6. Whenever an interface receives an IPv6 address (whether by SLAAC, DHCPv6, or manual entry), it sends a Neighbor Solicitation message out to the network asking, "Does anyone have this IP?" If someone replies, the address is marked as a duplicate and disabled.

  • A. SLAAC: (Incorrect) The process of generating an IP, not checking it.
  • B. DAD: (Correct) The verification process ensuring IP uniqueness.
  • C. EUI-64: (Incorrect) A method of generating an interface ID.
  • D. ARP: (Incorrect) IPv4 protocol.
11. What are two characteristics of the SLAAC method for IPv6 address configuration? (Combined)
Correct Answer: The default gateway will be the link-local address of the router AND IPv6 addressing is dynamically assigned through ICMPv6.

Deep Dive: SLAAC is totally reliant on ICMPv6. The router sends an RA (Router Advertisement) containing the network prefix. The host uses this to assign itself a Global Unicast Address. Simultaneously, the host takes the source address of that RA message (which is the router's Link-Local Address) and uses it as its default gateway.

  • B. Stateful / Clients send RA: (Incorrect) SLAAC is stateless. Routers send RAs, clients send RSs.
  • C & D. Routers send Solicitations: (Incorrect) Hosts send Solicitations (RS) to ask for information. Routers send Advertisements (RA) to give the information.
  • A. Link-local gateway & ICMPv6 assignment: (Correct) Accurately describes the core functions of SLAAC.
12. After booting, a client receives an ICMPv6 RA message with the M flag set to 0 and the O flag set to 1. What does this indicate?
Correct Answer: The client should automatically configure an IPv6 address and then contact a DHCPv6 server for more information.

Deep Dive: RA messages have specific flags to instruct clients. The M-flag (Managed) controls the IP address. The O-flag (Other) controls things like DNS.

M=0 means "Do not use Stateful DHCPv6 for your IP, use SLAAC instead."
O=1 means "Go to the Stateless DHCPv6 server to get Other info (like DNS)."

  • A. Statically configured: (Incorrect) The M and O flags are for dynamic setups.
  • B. Without contacting a server: (Incorrect) This would be M=0, O=0 (Pure SLAAC).
  • C. Configure address, then contact server for more info: (Correct) This perfectly describes M=0, O=1 (Stateless DHCPv6).
  • D. Request IP directly from server: (Incorrect) This describes M=1 (Stateful DHCPv6).
13. A network administrator is entering the command ipv6 unicast-routing to start configuring DHCPv6 operation on a router. Which statement describes the function of this command?
Correct Answer: It is required for sending ICMPv6 RA messages.

Deep Dive: By default, Cisco routers have IPv6 routing completely disabled! If a router has IPv6 routing turned off, it is physically impossible for it to broadcast Router Advertisement (RA) messages. Without RAs, clients cannot use SLAAC or find the DHCPv6 servers. The command ipv6 unicast-routing awakens the IPv6 brain of the router, allowing it to send RAs.

  • A, B, C. Required for DNS / DHCPv6 servers: (Incorrect) While you often type this command before setting up DHCPv6, its specific technical function is enabling the router to act as an IPv6 gateway and broadcast RA messages.
  • D. Required for sending RA messages: (Correct) Enabling IPv6 routing is the trigger that starts the periodic broadcasting of ICMPv6 RAs.
